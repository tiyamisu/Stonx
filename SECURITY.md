# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Yes    |

## Reporting a Vulnerability

StonX is an **educational simulation project** — it does not handle real money or personal financial data. However, if you discover a security vulnerability, please:

1. **Do NOT** open a public GitHub issue
2. Email the maintainer directly or use [GitHub's private vulnerability reporting](https://github.com/tiyamisu/Stonx/security/advisories/new)
3. Include a clear description and reproduction steps

We will acknowledge the report within **48 hours** and provide a fix timeline.

## Notes

- All trading data is **simulated** and stored locally in flat files
- No real financial APIs or payment systems are used
- Default credentials (`java_demo / 1234`) are for demonstration only — do not use in production
